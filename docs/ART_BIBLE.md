# 像素美术规范 v0.2

> 依赖：[世界圣经](./WORLD_BIBLE.md)  
> 目标：使用 GPT-Image 2 与像素编辑工具生产一致、可动画、可长期扩展的原创资产

## 1. 美术目标

画面需要同时传递：

- 私人小屋的亲密与安全。
- 边境世界的神秘与辽阔。
- 主角和伙伴值得长期陪伴的生命感。
- 比当前 CSS 几何图形更高级的像素细节与动画表现。

可以研究经典怪物收集游戏的轮廓辨识、温馨生活游戏的环境叙事和像素动画节奏，但不得复制角色、建筑、图块、调色板、UI 布局或可识别的商业资产。

## 2. 基础视觉规则

- 视角：3/4 俯视，角色与建筑保持同一透视。
- 地图基础格：32×32 px。
- 主角标准单帧：32×48 px；所有装备层共享该帧尺寸与原点。
- 小型伙伴标准单帧：32×32 px；大型生灵单独定义整数倍画布。
- 物品图标：24×24 px 或 32×32 px，同一界面不混用视觉比例。
- UI 图标：16×16、24×24、32×32 三档。
- 光源：默认左上方；室内允许壁炉作为第二暖光源。
- 轮廓：1 px 深色轮廓，局部可使用彩色轮廓，不使用纯黑包围全部物体。
- 边缘：禁止抗锯齿、模糊和非整数像素。
- 缩放：运行时只使用整数倍缩放并启用 `image-rendering: pixelated`。
- 动画：优先 6 至 10 fps 的手工像素节奏，不追求 60 fps 补间。

### 首个美术垂直切片

- 逻辑视口：320×180 px，运行时仅以整数倍最近邻放大。
- 场景：冬夜的边境小屋，只包含主角、小狗与可交互家具。
- 目的：先验证角色、环境、光照、比例和亲密感能否组成同一个可游玩世界。
- 主角风格锚点：`assets/art/reference/protagonist-style-key-v1.png`。
- 小屋环境风格锚点：`assets/art/reference/cottage-style-key-v1.png`。
- 生成提示词：`assets/art/prompts/cottage-vertical-slice-v1.md`。
- 风格锚点只确认情绪、轮廓、比例、空间构图和色彩方向，不得直接作为正式精灵图或地图背景使用。
- 小屋锚点的入口、人物融合与归家氛围已通过；原图尺寸、色数、像素网格、分层与交互性未通过生产标准。
- 小屋切片未通过前，不批量生产城镇、松风林、古龙、居民或装备资产。

## 3. 主调色板

首个切片建立一套 32 至 40 色的共享主调色板，包含：

- 暖木棕与壁炉橙。
- 麦穗金与柔和羊皮白。
- 苔藓绿、松林深绿与河谷青绿。
- 石墙灰、远山灰蓝与冬夜深蓝。
- 少量用于稀有魔法、星辉和天使的高亮色。

角色可以拥有局部专属色，但必须能够融入共享环境。稀有物品不能仅靠高饱和金色表达稀有度。

## 4. 主角分层

主角不能导出为一张固定图片。首版使用以下图层：

1. 身体与肤色。
2. 头发后层。
3. 基础服装。
4. 铠甲或外套。
5. 鞋与手套。
6. 头发前层或头饰。
7. 武器、盾牌和背包。
8. 披风前后层。

所有图层必须共享相同原点、帧数和命名。装备系统才能让宝剑和铠甲真实显示在主角身上。

### 首版动作

- 四方向待机。
- 四方向行走。
- 坐下。
- 阅读。
- 睡觉。
- 抚摸伙伴。
- 打开宝箱。
- 庆祝。
- 基础拔剑与受击姿态（战斗后续实现，资产可预留）。

## 5. 伙伴与坐骑

### 伙伴

- 小型伙伴标准单帧为 32×32 px；头像或档案像素肖像可使用 64×64 px。
- 每个伙伴必须具备可在 32×32 剪影中识别的核心身体符号；不得只依赖大眼睛、毛茸茸、大耳朵或长尾巴区分。
- 首发统一只生产四类动画：`idle`（4 帧）、`walk`（四方向、每方向 4 帧）、`sleep`（3 帧）、`growth_transition`（5–6 帧）。
- 不为单个伙伴制作天气、返航、读信、专属性格或复杂行为树动画；环境与个体差异通过基础动画、文本互动、场景位置与记忆描述表达。
- 栗子是第一个生产模板：犬型低重心、栗色毛与炉灰胸斑、小旧铜铃和褪色布结；其完整资产与生态位不得被后续伙伴复制。
- 不能继续使用所有物种共享的 CSS 方形身体。

### 年轻古龙

- 保留幼年亲近感，但轮廓必须暗示成年后的体量与古老血脉。
- 动作：待机、行走、短距飞行、睡眠、警觉、吐出微小火星、观察遗迹。
- 第一版不制作成年古龙完整动画，只制作远景剪影和极少数关键帧。

### 马匹

- 马体、鬃毛、毛色、马鞍、缰绳和行囊分层。
- 动作：待机、行走、小跑、进食、被梳理。
- 主角骑乘帧必须与马匹原点严格对齐。

## 6. 环境资产

### 小屋

- 木墙、石基、地板、窗、门、壁炉、床、书桌、宝箱、书架和小狗睡垫。
- 昼夜与四季通过窗外、光照、摆件和局部覆盖层变化，不为每季复制整张场景。
- 家具允许后续摆放，地图必须采用可占格数据而非一张不可交互背景图。
- 基础建筑图集合同：`assets/art/specs/cottage-architecture-v1.json`。
- GPT-Image 2 草稿提示词：`assets/art/prompts/cottage-architecture-tileset-v1.md`。
- 基础建筑图集为 256×256 逻辑像素、8×8 格、每格 32×32；家具、角色和光照不得混入该图集。
- 已通过的首个生产切片：`assets/art/environments/cottage/cottage_floor_wood_v1.png`，包含 5 个 32×32 横向木地板变体。
- 已通过的素墙生产切片：`assets/art/environments/cottage/cottage_wall_plain_v1.png`，包含 5 个 32×32 无立柱墙面变体；结构梁必须作为独立覆盖层。
- 独立结构层已包含 2 个 32×32 木梁覆盖帧、3 个 64×32 内墙角和 4 个 32×32 门口状态；门口开放帧使用硬透明显示外部场景。

### 城镇

- 石板路、木梁墙、石墙、坡屋顶、花盆、木牌、提灯、摊位和城门。
- 第一版只制作旅店外观/内景、小屋、制图室门面和一个城镇广场。

### 松风林

- 松树、灌木、草、药草、莓果、倒木、路牌、泥土路、石块与营地物件。
- 春夏秋冬使用覆盖层和调色变体。
- 第一次远征需要倒下路牌、岔路和遗失包裹专用资产。

## 7. GPT-Image 2 生产流程

GPT-Image 2 的产物是受控的美术输入，不直接视为可发布 spritesheet。

1. 先生成角色概念页，确定轮廓、比例、服装和主色。
2. 选定概念页作为后续所有变体的唯一参考图。
3. 单独生成角色转面、表情与动作参考。
4. 按统一网格生成 spritesheet 草稿。
5. 在 Pixelorama、LibreSprite 或同等工具中逐帧清理。
6. 运行资产检查，验证尺寸、透明边缘、帧数、命名和调色板。
7. 导出 PNG 图集与 JSON 元数据。

禁止为同一角色每次使用全新文字提示而不带参考图。那会造成脸型、服装、比例和色彩漂移。

## 10. 资产自动检查

构建前检查：

- 图片尺寸是否为声明网格的整数倍。
- 是否存在半透明脏边和意外抗锯齿。
- 同一动作的所有图层帧数是否一致。
- 原点和碰撞脚点是否一致。
- 是否使用未登记颜色或异常高色数。
- 图集是否存在重名帧和缺失元数据。
- CSS/Canvas 是否使用整数倍缩放。

项目使用 `assets/art/manifest.json` 区分参考图和生产资产。新增生产 PNG 时必须登记帧尺寸、行列、透明规则和色数上限，并运行：

```bash
npm run art:validate
```

正式构建会通过 `prebuild` 自动执行同一检查。任何 GPT-Image 2 原图都默认属于参考或草稿，只有经过像素清理、分层、登记并通过检查后才能进入运行时图集。

### 临时运行角色的边界

`player_idle_front_provisional_v1.png` 只用于验证 32×48 主角在 320×180 场景中的占比、脚点、移动速度、碰撞与环境融合。它不是角色定稿，不得据此宣称以下项目已经完成：

- 四方向站立与行走动画；
- 身体、发型、披风、服装、武器和背包分层；
- 北向、侧向动作或战斗姿态；
- 可直接发布的最终角色图集。

正式替换时必须保持相同逻辑尺寸和脚点，使场景代码无需因美术升级而重写。

### 临时完整场景层的边界

`cottage_room_backdrop_provisional_v1.png` 是从已确认的环境风格锚点中生成的 320×180、有限色板运行时场景层。它用于快速验证“这是否像一个愿意回来的家”、空间比例、主角融合、家具碰撞与出口位置，不代表环境资产分层已经完成。

- 背景中只保留家具；主角和小狗均已从背景移除，由运行时独立叠加；
- 初始炉边小猎犬拥有 32×32 临时精灵，其他伙伴当前使用可替换的有限色板占位轮廓，正式上线前仍需各自的独立四向图集；
- 壁炉闪烁由运行时光照层承担，背景本身仍含概念阶段的烘焙明暗；
- 书桌、床铺、壁炉、宝箱、盆栽和长凳已有脚点碰撞，但尚不能逐件使用；
- 后续必须按照用户体验优先级，把高频交互物件逐件替换为独立资产，而不是一次性重做整张图。

该层不得扩展为其他房间模板，也不得作为城镇、旅店或居民住宅的通用背景。

## 11. 第一轮 GPT-Image 2 概念提示词骨架

```text
原创温暖中世纪幻想像素美术角色设定页，3/4 俯视游戏视角。
角色是一名友善、善良、乐于助人的年轻边境旅行者，穿朴素旅行服、短披风、皮革背包，携带使用多年的旧旅行剑。
整体气质亲密、可靠、充满好奇心；不是王族、天选英雄或黑暗战士。
使用有限暖色调色板、清晰像素簇、硬边缘、无抗锯齿、左上方光源。
展示正面、背面、左右侧面、三种温和表情与基础行走姿态。
保持各视图服装、比例、发型和装备完全一致。
纯色背景，无文字、无 UI、无现有游戏角色或可识别商业设计。
```

概念确认后再为主角、小狗、小天使和环境分别编写严格的 spritesheet 提示词，不能在角色尚未定稿时批量生产全部资产。
