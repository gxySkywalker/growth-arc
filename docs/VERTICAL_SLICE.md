# 第一次远征垂直切片 v0.1

> 依赖：[世界圣经](./WORLD_BIBLE.md)  
> 目标：用一条完整可玩的体验证明“真实专注会让私人中世纪世界永久生长”

## 1. 范围

垂直切片包含：

1. 可跳过、可重看的抵达城镇开场动画。
2. 主角取名与基础外观创建。
3. 入住小屋并看到主角与小狗共同生活。
4. 莉娅留下第一份松风林地图委托。
5. 用户绑定一个真实任务并开始专注。
6. 主角与小狗在松风林远征的安静专注画面。
7. 完成后解锁第一段道路、宝藏与遗失包裹事件。
8. 只记录“本次完成了什么”的轻量结算。
9. 返回小屋并收到小天使第一封信。

## 2. 明确不在本切片内

- 完整王国地图。
- 完整回合制战斗。
- 正式恋爱、婚姻和多人同行者系统。
- 所有伙伴、坐骑、季节活动与商队内容。
- 完整飞书同步。
- 一次实现上百件物品或大量程序化剧情。

切片需要为这些系统留下数据接口，但不能假装已经完成。

## 3. 核心验收承诺

用户完成第一次专注后，必须同时感受到：

- 我本人出现在世界里，而不是只看见一只宠物。
- 小狗真的与我共同出发和返家。
- 现实任务和远征目的存在明确联系。
- 专注期间没有游戏操作打扰。
- 地图因为我的行动永久多出一段道路。
- 带回的物品可以实际使用、交换、制作、摆放或推动故事。
- 结算没有强迫我规划下一步或写长笔记。
- 我愿意为了看看下一次会发生什么再次打开软件。

## 4. 用户状态兼容

### 新用户

- 首次启动进入开场动画。
- 动画后创建主角。
- 创建主角后进入第一天傍晚的小屋。

### 已有用户

- 保留现有任务、专注记录、伙伴、背包、设置和 AI 报告。
- 自动创建待完善的主角档案，不清空数据库。
- 首页温和提示“为旅途中的自己补上形象”，用户可以稍后处理。
- 开场动画可以主动观看，不能强制覆盖当前使用流程。
- 现有栗子继续作为最初小狗伙伴。

## 5. 场景流程

### 场景 A：黄昏旧路

**画面**

- 3/4 俯视像素场景。
- 主角背着行囊从画面下方走向远方城门。
- 小狗跟随，远处窗灯依次亮起。
- 画面根据真实季节和时间选择颜色变体。

**交互**

- 允许跳过。
- 首次播放无复杂按钮。
- 完成后标记 `intro_seen_at`。

### 场景 B：旅店与第二天清晨

**画面**

- 奥伦端来热汤，小狗在壁炉边睡下。
- 次日由艾达带主角来到小屋。

**文本原则**

- 总对白短于 180 个汉字。
- 不一次解释王国历史。
- 不使用天选、预言、灾难或强制使命。

### 场景 C：创建主角

**首版可选项**

- 名字。
- 体型基础层。
- 肤色。
- 发型与发色。
- 初始旅行服颜色。
- 披风显示开关。

**规则**

- 所有选项可在小屋内再次修改。
- 不询问复杂职业和永久属性。
- 创建结果保存为分层 sprite 配置，而不是一张不可修改图片。

### 场景 D：入住小屋

**必须出现**

- 主角、小狗、床、壁炉、书桌、宝箱和窗外真实时间。
- 永远可见的一键“开始远征”。
- 当前任务与今日准确专注数据。
- 小屋作为默认全尺寸世界页；统计、任务与背包明细进入独立“旅程总览”。
- 常伴伙伴由当前数据动态渲染，不允许把某只伙伴烘焙进房间背景。
- 主角与常伴伙伴必须具有独立脚点、碰撞和纵深排序；靠近后通过统一键盘交互键交谈，不能只是互相覆盖的装饰图片。

**禁止**

- 用十几个红点引导用户逐一领取。
- 把所有功能藏进场景热点。
- 主角或伙伴因离线表达失望。

### 场景 E：莉娅来访

莉娅说明松风林旧路多年无人确认，并留下一张地图。该事件只创建世界委托，不自动替用户创建现实任务内容。

用户仍然选择自己的真实任务；系统将其绑定为本次“确认旧路”的现实目标。

### 场景 F：远征准备

**输入**

- 现实任务或自由输入的本次完成内容。
- 计划专注时长。
- 本次同行伙伴（默认栗子或当前常伴伙伴，但保留独立选择控件）。
- 可选食物或工具（切片允许为空）。

**输出预览**

- 目的：确认松风林旧路。
- 地区：松风林入口。
- 天气与建议衣着。
- 当前队伍。

用户一次确认即可开始，不要求配置全部项目。

### 场景 G：专注远征

**画面**

- 主角与小狗同时出现并沿森林旧路缓慢前进。
- 计时、暂停、结束与当前任务清晰可见。
- 根据真实季节、时间和虚构天气切换场景。
- 可以切换为低动态或纯计时模式。

**行为**

- 专注期间不弹战斗、对话选择、宝箱或商队。
- 动画不能造成显著 CPU/GPU 占用。
- 系统休眠、崩溃与重启继续遵守现有会话恢复规则。

### 场景 H：返程发现

第一次完成远征固定创建以下事实：

1. 松风林入口至倒下路牌之间的道路被发现。
2. 道路可通行，但存在一条尚未探索的岔路。
3. 小狗发现一个遗失包裹。
4. 用户获得一件基础可用物品。
5. 莉娅的关系从“初识”变为“认识”。

文案可以有小范围变化，但 AI 不得改变以上事实。

### 场景 I：轻量结算

**显示顺序**

1. 完成时长与现实任务。
2. 地图新道路动画。
3. 遗失包裹和宝藏。
4. 一行可选输入：“本次完成了什么”。
5. 返回小屋。

**禁止出现**

- “下一步准备做什么”。
- 强制学习感想。
- 冒险日志编辑器。
- 因未填写而减少奖励。

### 场景 J：第一封信

返回小屋后的当晚，窗边出现无署名信。第二天小天使正式到访。

第一封信必须：

- 只引用真实的任务、时长和完成记录。
- 不夸张声称用户掌握了没有记录的知识。
- 祝贺第一次真实远征。
- 附带一件不可误删的纪念品。

## 6. 世界状态最小模型

切片实现前需要将以下状态从 UI 文案中抽离：

### 玩家

- `player_profile`：名字、外观层、创建时间、开场状态。
- `player_skills`：先建立可扩展接口，切片不开放完整成长树。

### 地图

- `world_regions`：固定区域定义。
- `world_nodes`：道路、路标、地点和发现状态。
- `world_edges`：节点之间的路线。
- `discoveries`：何时、由哪次远征解锁了什么。

### 内容

- `content_version`：规则与内容包版本。
- `event_definitions`：事件允许条件与固定结果。
- `event_log`：实际发生的唯一事件，防止重复初遇。

### 关系

- `characters`：居民与同行者定义。
- `relationships`：关系阶段、关键事件与承诺。

### 物品

- `item_definitions`：分类、稀有度、动作和保护规则。
- `inventory_transactions`：所有获得、消耗、交换与制作记录。

现有表必须通过增量迁移兼容，不能重建或清空用户数据库。

## 7. 奖励曲线要求

现有按 5、15、25、45、60、90 分钟硬跳档的逻辑需要在切片阶段替换或版本化。

- 地图微发现每次完成都有。
- 稀有概率使用连续、递增且逐渐衰减的曲线。
- 90 分钟后不再提高稀有概率。
- 不鼓励为了跨过一分钟边界继续硬撑。
- 战斗、笔记和结算填写不影响基础专注奖励。
- 旧远征结果保持不变，新算法必须带版本号。

## 8. 物品切片要求

当前 9 件物品不能继续以硬编码 `if/else` 作为最终模型。每件物品定义至少一个合法动作：

| 物品 | 第一用途 |
| --- | --- |
| 旧王朝铜币 | 城镇交易或商队补差 |
| 手绘地图碎片 | 拼合或选择后续路线 |
| 山野药草 | 烹饪、制作或赠送 |
| 莓果旅行面包 | 分享并触发伙伴互动 |
| 蜜色琥珀碎片 | 制作灯饰、挂件或装备 |
| 月银罗盘 | 预览远征地点或事件范围 |
| 古龙鳞片 | 进化仪式或高阶制作 |
| 星辉玻璃 | 天文台观察与稀有事件线索 |
| 旅行者银铃 | 推进新伙伴相遇或召来访客 |

物品详情提供使用、赠送、合成、摆放、交换和查看故事等上下文动作。不可用动作应被隐藏或解释条件，不能在点击后才报“暂时无法使用”。

## 9. 技术拆分

建议实现顺序：

1. 内容包格式、版本号和数据迁移骨架。
2. 主角档案与分层 sprite 渲染器。
3. 地图节点、路线和发现记录。
4. 新首页小屋场景与老用户兼容入口。
5. 第一次远征事件状态机。
6. 主角与小狗的专注场景。
7. 轻量结算与第一封信。
8. 宣言式物品动作与连续奖励曲线。
9. 测试、性能与打包验证。

## 10. 测试与验收

### 自动测试

- 旧数据库迁移后记录数量和关键字段不减少。
- 第一次远征事件只能完成一次。
- 相同结算请求不会重复发放道路、物品和信件。
- 地图发现可以追溯到唯一会话。
- 未填写完成记录不会减少奖励。
- 90 分钟后的稀有概率不继续增长。
- AI 不可修改固定事件结果。
- 离线模板可以完整完成第一次远征。

### 体验验收

- 从打开软件到开始远征不超过三次主要点击（已完成角色创建后）。
- 开场可以跳过，且不阻塞老用户。
- 专注画面同时看见主角与小狗。
- 返回后 90 秒内可以完成全部结算。
- 所有新增动画提供减少动态选项。
- 1366×768 和 1920×1080 下主要操作均可见。

## 11. 完成定义

只有同时满足以下条件，垂直切片才算完成：

- 世界状态不是临时前端状态，重启后仍存在。
- 老用户数据完整迁移。
- 第一条道路、宝藏、关系和信件具备幂等记录。
- 主角真正出现在小屋、专注与返程中。
- 至少一套经过清理的真实 spritesheet 替代 CSS 方块主角/伙伴。
- 核心流程无 API Key 也可完整使用。
- 构建、测试和 Windows 安装包通过。
