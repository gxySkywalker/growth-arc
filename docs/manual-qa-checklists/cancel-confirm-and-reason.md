# 手动验收清单：远征 → 返程清点 → 空手返回确认 → 远征归来 → 编年史

## 环境准备

1. 启动 `npm run dev`
2. 打开 DevTools：`Ctrl + Shift + I`，切换到 Console 标签
3. 打开主进程终端（查看后端日志）
4. 分辨率：**1366×768**，系统缩放 **125%**

---

## TC-01：正常结算 — 主要路标 + 沿途 → 远征归来 → 编年史

**前置**：已有至少 2 个活跃路标（例如"看热爱1"和"刻板"）

**步骤**：
1. 点击"开始远征" → 选择"看热爱1" → 计划 15 分钟 → 穿过城门
2. 等待约 10 秒 → 点击"准备返航" → 确认主要路标 ✓ → 勾选"刻板"作为沿途
3. 填写 outcome："完成了习题"
4. 点击"带着成果回小屋"

**UI 验证**：
- [ ] 返程清点关闭，远征归来弹窗出现
- [ ] 主要抵达："看热爱1" + 状态行"路标已完成"（短时）
- [ ] 沿途抵达："刻板" + 状态行"行程较短"
- [ ] 掉落区：至少 1 件普通掉落，标题无"本次带回"（仅有"本次收获"）
- [ ] 一页完整显示，无滚动条
- [ ] Console 无红色报错

**数据库验证**（`npm run dev` 终端或 sql.js 查询）：
- [ ] `focus_sessions.status = 'completed'`
- [ ] `session_task_links` 中"刻板"的 `reason = 'short_session'`，`xp_awarded = 0`
- [ ] `expeditions` 表有对应记录

**编年史验证**：
- [ ] 编年史时间轴显示"看热爱1"，含掉落
- [ ] 展开完整档案：`实际专注 … 秒`（非"实际 0 分钟"）
- [ ] 沿途："刻板" 显示 `已完成 · 行程较短`
- [ ] 若有 `task_completed=1`：显示"已抵达路标"

## TC-02：空手返回 — 鼠标完整流程

**前置**：正在远征中

**步骤**：
1. 开始远征 → 等待 5 秒 → 点击"准备返航"
2. 点击左下"空手返回"按钮
3. 确认弹窗出现
4. 点击遮罩（弹窗外暗色区域）

**UI 验证**：
- [ ] 确认弹窗关闭，返程清点保持打开
- [ ] outcome、主要路标勾选、沿途选择全部保留
- [ ] 焦点回到"空手返回"按钮

**步骤（续）**：
5. 再次点击"空手返回"
6. 点击右上角 X（已隐藏，CSS 确保不可见）
7. 点击"继续清点"

**UI 验证**：
- [ ] 确认弹窗关闭，返程清点保持打开
- [ ] 所有状态保留

**步骤（续）**：
8. 再次点击"空手返回" → 点击"空手返回"（确认按钮）

**UI 验证**：
- [ ] 确认弹窗关闭，返程清点关闭
- [ ] 回到炉火小屋
- [ ] 无"远征归来"弹窗
- [ ] Console 无红色报错

**数据库验证**：
- [ ] `focus_sessions.status = 'cancelled'`
- [ ] `focus_sessions.active_seconds > 0`
- [ ] 无 `expeditions` 记录
- [ ] 无 `xp_transactions` 记录

## TC-15：数据库 reason 持久化端到端

**前置**：全新测试环境或可清理的开发数据库

**步骤**：
1. 创建一个活跃路标（不预先完成过）
2. 开始一次 < 5 分钟的远征
3. 在返程清点中勾选该路标作为沿途
4. 点击"带着成果回小屋"

**数据库验证**：
```sql
SELECT l.task_id, l.reason, l.xp_awarded, s.active_seconds
FROM session_task_links l
JOIN focus_sessions s ON s.id = l.session_id
WHERE l.session_id = '<最新 session_id>';
```
- [ ] `reason = 'short_session'`
- [ ] `xp_awarded = 0`

**前端验证**：
- [ ] 远征归来弹窗中该沿途显示 `行程较短`
- [ ] 编年史中同一记录也显示 `已完成 · 行程较短`
