# 伙伴生产规则 v2.0（冻结）

> 依赖：[世界圣经](./WORLD_BIBLE.md) · [美术规范](./ART_BIBLE.md)  
> 状态：冻结。适用于未来 8 个、16 个及更多伙伴；新增伙伴不得绕过本规则。

## 1. 核心定位

伙伴不是宠物模拟器、战斗单位、数值装备、自动生产工具或效率提升工具。

伙伴是：

> 在这个世界中生活，并与旅人共享一段时间的朋友。

伙伴体验遵循：

```text
家 → 出发 → 同行 → 经历 → 成长 → 被记住
```

设计参考的是怪物收集游戏中的相遇、收集、个体差异与成长；温暖生活游戏中的日常陪伴、家的归属与时间积累；以及天使邮局对真实共同经历的保存。不得复制任何既有商业角色、名称、形象或素材。

## 2. 生产模型

每个伙伴由 `Species` 与 `Companion Instance` 组成。

### Species（物种）

物种定义世界中的共同内容：

- 名称与外观；
- 栖息环境；
- 核心身体特征；
- 核心动作；
- 情感关键词；
- 成长路线；
- 专属生活习惯池；
- 叙事特性；
- 32×32 sprite、64×64 头像与四类动画资源。

### Companion Instance（玩家伙伴个体）

个体定义玩家实际同行的那一只：

- 昵称；
- 轻微外观变体；
- `personalityProfile`；
- 羁绊与成长阶段；
- 共同记忆。

目标是让玩家感到“这是我的那一只”，但不将个体差异转化为强度、收益或最优解。

## 3. 个体差异：personalityProfile

每个伙伴实例恰好拥有：

```text
personalityProfile
├── personalityTrait  通用性格印象，1 个
├── habit             该物种专属生活习惯，1 个
└── quirk             通用小毛病，1 个
```

### personalityTrait：通用性格印象

所有伙伴共用同一池，每个个体随机获得 1 个：

- 活泼、安静、慢热、胆小、温和、好奇、独立；
- 倔强、爱观察、爱凑热闹、认真、有耐心、贪玩、谨慎。

### habit：物种专属生活习惯

每个物种拥有自己的习惯池。习惯是生活细节，不是缺点，也不产生额外系统。

例如，栗子的习惯池可以包括：喜欢靠近炉火、喜欢闻旅人带回的旧物、喜欢把小物件放在角落、喜欢趴门槛晒太阳。

### quirk：真实的小毛病

小毛病必须是轻微但真实的性格摩擦，不是卖萌标签。

可用示例：有点怕雷声、对陌生声音敏感、不喜欢改变路线、容易担心旅人、有点固执。

不可用示例：可爱、爱吃、喜欢玩。这些应归入性格或习惯。

### 个体差异的唯一影响范围

`personalityProfile` 只影响：

- E 键互动文本；
- 编年史描述；
- 共同记忆文字。

它不得影响：

- 数值与羁绊获取；
- 成长速度或成长结果；
- 动画与场景位置；
- 获取概率、稀有度；
- 远征收益、掉落或效率。

## 4. 动画资产生产规范

伙伴不是动画模拟系统。所有伙伴首发统一只制作以下四类动画：

| 动画 | 帧数 | 用途 |
| --- | ---: | --- |
| `idle` | 4 | 小屋与场景等待 |
| `walk` | 四方向、每方向 4 | 跟随与移动 |
| `sleep` | 3 | 夜晚与小屋休息 |
| `growth_transition` | 5–6 | 成长形态变化 |

禁止按伙伴单独制作天气动画、回家动画、专属迎接动画、专属性格动画、读信动画或复杂行为树。

天气、时间、读信、返航和互动统一通过：

```text
基础动画 + 文本互动 + 场景位置 + 记忆描述
```

表达。

## 5. 成长系统

成长不是强化；成长代表伙伴关系和陪伴方式的变化。

统一规则：每累计 100 点羁绊进入一个阶段。

```text
0–99：初始形态
100–199：成长形态
200：最终形态
```

最终形态由“羁绊达到 200 的成长结算时间”决定：

| 时间 | 最终形态 |
| --- | --- |
| 06:00–15:59 | 形态 A |
| 16:00–19:59 | 形态 B |
| 20:00–05:59 | 形态 C |

成长结果一经确定永久保存，不重新成长。形态不得影响属性、掉落、效率或强度。

### 成长触发与持久化

羁绊的来源可以是共同远征，也可以是明确写着“与伙伴分享”的背包物品。无论来源为何，只要一次结算使羁绊**首次跨过** `100` 或 `200`，系统都必须创建一条持久的成长记录，并播放同一套成长仪式。

- 成长记录在仪式完整观看前不得丢弃；关闭程序后，下一次进入伙伴营地仍须继续呈现；
- 同一伙伴、同一成长阶段只播放一次；
- 物品分享是温暖互动，不是每日强制喂养、饥饿值或离线惩罚。

### 成长仪式美术规则

成长仪式借鉴 GBA 时代怪物收集 RPG 的**演出语法**，不复制任何既有角色、素材、音乐、音效、名称或具体台词。

固定流程：

```text
伙伴变化提示 → 旧形态白色剪影 → 旧/新轮廓节奏性交替与像素闪光 → 新形态稳定显现 → 记录这一刻
```

- 画面只能使用该伙伴已经生产的形态 sprite、基础光效、短促原创提示音和项目自有文字；
- 白色剪影必须是现有角色的滤色轮廓，不能补画或生成未定义形态；
- 节奏应短、清晰、可跳过；完成后由玩家确认“记下这一刻”；
- 成长不是奖励结算页面，不显示属性、强度、战力、稀有度或效率比较；
- 对栗子，第一段仪式文案以“栗子的身体，好像发生了一些变化……”开场，以“炉尾长成了栗鬃。”收束；最终形态使用同一语法与实际冻结形态名。

## 6. 每位伙伴的设计门槛

任何新伙伴必须明确回答：

1. 核心身体符号是什么？
2. 核心动作是什么？
3. 它代表什么情感？
4. 与已有伙伴有什么视觉区别？
5. 是否能在 32×32 像素剪影中识别？
6. 是否能只用四类动画完成？
7. 是否可作为内容配置扩展到 16 个以上伙伴？

每位伙伴必须占据不同的：

```text
身体符号 + 动作语言 + 环境联系 + 情感关键词
```

不得复制既有伙伴的完整生态位。

## 7. 栗子：已冻结的第一模板

| 维度 | 冻结内容 |
| --- | --- |
| 名称 / 种族 | 栗子 / 炉尾 |
| 定位 | 玩家最初旅行伙伴；与玩家一同抵达边境小镇 |
| 身体 | 犬型低重心、方口鼻、厚实前爪、略长身体、短厚尾巴 |
| 视觉 | 栗色毛发、炉火色调、小旧铜铃、褪色布结 |
| 动作 | 闻路、等待、回头确认 |
| 环境 | 炉火小屋、旧路、归途 |
| 情感 | 被等待的归处 |
| 动画 | idle、walk、sleep、growth_transition |
| 成长 | 炉尾 → 栗鬃 → 炭尾 / 松影 / 月爪 |

未来伙伴不得复制栗子的犬型低重心、旅行铜铃、熟路鼻、旧路与归途主题，或“被等待的归处”这一情感生态位。
